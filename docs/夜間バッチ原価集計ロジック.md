# 夜間バッチ原価集計ロジック

夜間バッチは、日中の業務処理への負荷を軽減するため、**夜間に計算ロジックを実行し、結果を専用の集計テーブルに格納**する仕組みです。ソース資料に基づく構成・紐付け・集計区分をまとめます。

**実装**: `sql/night-batch-cost-aggregation.sql`（PL/pgSQL ストアドプロシージャ）

---

## 1. 集計の基本原則（キーの紐付け）

工事番号は数年で巡回して再利用されるため、**単一の「工事番号」では一意にならず、必ず以下の複合キー**でデータを特定・集計します。

| キー | 説明 |
|------|------|
| **ConstructionNo（工事番号）** | 案件を識別する番号。 |
| **KeyDate（受注登録日）** | 受注登録日。工事番号と組み合わせて一意性を保つ。 |

- すべての集計は `GROUP BY ConstructionNo, KeyDate`（または `constructno, keydate`）でグループ化する。
- 参照・更新時もこの複合キーを条件に含める。

---

## 2. 主要な原価構成要素と集計ロジック

各要素は、それぞれの**実績テーブル**から「工事番号 ＋ 受注登録日」でグループ化し、**合計金額（SUM）**を算出します。**取消・削除フラグ**で有効なレコードのみを対象にします。

### ① 購入部品費の集計

設計指示に基づき発注・検収された市販部品の費用。

| 項目 | 内容 |
|------|------|
| **参照テーブル** | `t_purchaseparts` |
| **集計式** | `SUM(TotalPrice)`。TotalPrice が無い場合は `EachPrice × OrderQty` で計算。外貨の場合は `CurrencyRate` を乗じて円貨換算。 |
| **除外条件** | `CancelFlg = 0 AND DeleteFlg = 0`（取消・削除を除外） |
| **別製作除く** | `BetsuSeisakuFlg` が立っている行は対象外とする。 |

```sql
SELECT ConstructionNo, KeyDate, SUM(TotalPrice) AS 購入部品費
FROM t_purchaseparts
WHERE (CancelFlg = 0 OR CancelFlg IS NULL)
  AND (DeleteFlg = 0 OR DeleteFlg IS NULL)
  AND (BetsuSeisakuFlg = 0 OR BetsuSeisakuFlg IS NULL)
GROUP BY ConstructionNo, KeyDate;
```

---

### ② 外注加工費・材料費の集計

部品製作のための外注加工費（製缶、表面処理等）と素材購入費。

| 項目 | 内容 |
|------|------|
| **参照テーブル** | `t_manufctpurchase` |
| **判別** | 発注区分コード（`OrderDivCode`）で「外注加工費」と「材料費」を分けて集計。 |
| **外注加工費** | `OrderDivCode` が加工関連（例: C番）のレコードを合計。 |
| **材料費** | `OrderDivCode` が材料関連（例: R番）のレコードを合計。 |
| **除外条件** | `CancelFlg = 0 AND DeleteFlg = 0` |
| **別製作除く** | `BetsuSeisakuFlg` が立っている行は対象外。 |

---

### ③ 諸経費の集計

輸送費、梱包費、その他案件に付随する費用。

| 項目 | 内容 |
|------|------|
| **参照テーブル** | `t_expense` |
| **集計式** | `SUM(TotalPrice)` |
| **除外条件** | `CancelFlg = 0 AND DeleteFlg = 0` |
| **別製作除く** | `BetsuSeisakuFlg` が立っている行は対象外。 |

```sql
SELECT ConstructionNo, KeyDate, SUM(TotalPrice) AS 諸経費
FROM t_expense
WHERE (CancelFlg = 0 OR CancelFlg IS NULL)
  AND (DeleteFlg = 0 OR DeleteFlg IS NULL)
  AND (BetsuSeisakuFlg = 0 OR BetsuSeisakuFlg IS NULL)
GROUP BY ConstructionNo, KeyDate;
```

---

### ④ 社内工数（人件費）の集計

設計・加工・組立にかかった時間を金額換算。

| 項目 | 内容 |
|------|------|
| **参照テーブル** | `t_worktime`（機械単位）、`t_worktimekako`（図面単位） |
| **ロジック** | 作業時間（`WorkTime` / `hours`）を合計し、**部門ごとの労務費率**を乗じて金額化。 |
| **集計ビュー** | `v_工事番号別作業時間集計` — 機械設計、電気設計、社内加工、社内組立、電気艤装、試運転などの項目別に時間を集計。 |

---

## 3. 集計レベルと出力先テーブル

バッチ結果は、管理目的（粒度）に合わせて以下のテーブルに書き込みます。

| 集計レベル | 格納先テーブル（例） | 概要 |
|------------|----------------------|------|
| **工事番号別** | `t_工事番号別原価集計` | 案件全体の粗利益（売掛金 − 各原価）を算出。 |
| **機械別** | `t_機械別原価集計` | 同一工事内の機械ごとのコストを把握。 |
| **ユニット別** | `t_ユニット別原価集計` | 機械を細分化したユニット単位での集計。 |
| **図面番号別** | `t_図面番号別原価集計` | 個々の部品製作にかかった材料費・外注費の追跡。 |

---

## 4. 特殊な集計ロジック（別製作除く）

**別製作除く**とは、他の案件から流用した部品や、特定のフラグ（`BetsuSeisakuFlg`）が立っている工数・費用を、**対象案件の純粋な原価計算から除外**するためのフィルタです。

- 集計時に `WHERE (BetsuSeisakuFlg = 0 OR BetsuSeisakuFlg IS NULL)` を付与する。
- 該当フラグが存在しないテーブルでは条件を省略する。

---

## 5. まとめ

夜間バッチの SQL ロジックは、次の流れで構成されています。

1. **複合キー**: すべての集計で「工事番号 ＋ 受注登録日（KeyDate）」をキーに使用する。
2. **ステータス判別**: 有効レコードのみを対象とする（`CancelFlg = 0`, `DeleteFlg = 0`）。
3. **別製作除く**: `BetsuSeisakuFlg` が立っている行は集計対象から除外する。
4. **集約**: 購入部品費・外注加工費・材料費・諸経費・社内工数（人件費）をそれぞれ集計し、工事番号別・機械別・ユニット別・図面番号別の各集計テーブルへ一括で書き込む。

実装例は **`sql/night-batch-cost-aggregation.sql`** の PL/pgSQL を参照してください。

- **集計結果テーブル**: `t_cost_agg_by_construction`（工事番号別）、`t_cost_agg_by_machine` / `t_cost_agg_by_unit` / `t_cost_agg_by_drawing`（機械・ユニット・図面番号別の枠のみ定義。中身の集計は必要に応じて追加）。
- **実行**: `SELECT run_night_batch_cost_aggregation(3000);`（引数は人件費の時給単価）。
- 実績テーブルに `cancelflg`, `deleteflg`, `betsuSeisakuFlg`, `totalprice`, `orderdivcode` 等が無い場合は、ファイル先頭のコメントに従い ALTER TABLE で追加するか、既存スキーマに合わせて関数を修正してください。
