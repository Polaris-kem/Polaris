# 既存画面と業務の対応付け

Polaris の既存画面が、どの業務フロー・どの担当（営業・資材・明石・本社など）に対応しているかをマッピングしたドキュメントです。機能追加・画面改修時の参照用です。

---

## 1. 一覧マトリクス

## 2. 業務フロー別の対応画面

### 2.1 受注〜契約

- **受注登録** … 新規受注の入力・見積流用・プラスマージン。
- **受注データ（一覧）** … 一覧検索・納期確認。今後「希望納期」「実際出荷日」列を追加予定。
- **工事番号採番** … 受注前に工事番号を取得。

### 2.2 設計〜図面

- **図面番号採番** … 部品図・組立図の採番。設計データの起点。
- **加工進捗管理** … 図面番号・名称で進捗検索。色分け・紙出力との併用。

### 2.3 購買・資材

- **購入部品 納入処理** … A番号の納入日登録。発注前の在庫ポップアップと連携予定。
- **諸掛購入登録 / 諸掛納入処理** … FY的な経費・諸掛の購入と納入。
- **余り品登録 / 余り品検索** … 余剰在庫の登録と、発注時の「在庫あり」参照。
- （リストから）仕入・見積・発注関連テーブルをモーダル編集。

### 2.4 入金・請求

- 現状は**リスト画面で入金・会計テーブル**を参照・編集。**営業用の入金確認専用画面**は優先機能として要追加。
- 4分割（契約時／中間／検証／完納）の管理も要拡張。

### 2.5 出荷・物流

- **出荷管理** … 本社から客先への出荷。
- **外注・材料 納入処理** … 明石での外注品・材料の納入（毎日便で本社へ）。
- **工事番号出荷登録** … 工事単位の出荷まとめ。

### 2.6 製造・現場

- **作業票の登録** … 組み立て・加工の実績。
- **バーコードリーダー版 作業票** … 現場向けバーコード入力。

---

## 3. ギャップ・今後の拡張

- **希望納期と実際出荷日** … 受注・一覧に2種の納期を表示・登録する拡張が必要。
- **変更時メール** … 受注・金額・仕様変更のトリガーで自動メール送信を追加。
- **入金確認専用画面** … 営業が「入金されたか」だけ見る画面を新設またはダッシュボードから導線を追加。
- **発注前の在庫ポップアップ／列** … 購入部品納入処理または発注画面で、余剰在庫と照合して「在庫あり」を表示。
- **色分けの統一** … 加工進捗・受注一覧などで、水色／赤／黄／紫のルールを統一して適用。
